# 卫星地图视觉语言导航系统 - 设计思路

## 系统定位

工程系统，结合坐标导航 + 视觉语言理解，核心能力是**卫星地图匹配定位**。

---

## 整体架构

```
用户指令："找到xx建筑"
      ↓
大模型解析 → 触发 search_building skill
      ↓
skill: 调用地图API → 获取目标建筑卫星图（模板）
      ↓
无人机巡航（螺旋搜索）→ 俯视摄像头持续拍摄
      ↓
实时画面 vs 模板卫星图 → VLM 匹配判断
      ↓
匹配成功 → 飞过去 / 悬停
```

---

## 仿真环境

- 平台：Colosseum + UE4
- 地图：谷歌地图导入的三维真实地理地图
- 无人机：俯视摄像头 + 前视摄像头
- 坐标系：真实经纬度（WGS84）

## 关键模块

### 1. 指令解析 + Skill 触发

大模型接收自然语言指令，自动识别意图并触发对应 skill。

### 2. 地图API获取目标卫星图

```python
poi = map_api.search_poi(target_name)       # 查询目标经纬度
template = map_api.get_satellite_image(     # 获取卫星图模板
    poi.lat, poi.lon, zoom=18
)
```

### 3. VLM 视觉匹配

直接将无人机画面和卫星模板图同时传给 Qwen VL 进行比较：

```python
result = qwen_api(
    images=[drone_frame, satellite_template],
    prompt="""
    图1是无人机当前俯视画面，图2是目标建筑的卫星图模板。
    请判断：
    1. 图1中是否包含图2中的目标建筑？（是/否）
    2. 如果是，目标在图1的哪个方向？（左/右/前/后/中心）
    3. 匹配置信度 0-100

    返回JSON：{"found": true/false, "direction": "left", "confidence": 85}
    """
)
```

### 4. 搜索策略

```python
# 螺旋搜索
for radius in range(50, 500, 50):
    fly_circle(radius)
    if match_found():
        break
```

### 5. 导航控制闭环

```python
while not found:
    frame = drone.get_downward_image()
    result = match_with_vlm(frame, template)  # 低频，2-3秒一次

    if result["found"] and result["confidence"] > 80:
        drone.hover()
        found = True
    else:
        drone.move_slow(direction=result.get("direction", "forward"))

    time.sleep(2)
```

## 关键难点

| 难点 | 说明 | 应对 |
|------|------|------|
| 视角差异 | 卫星图正射 vs 无人机略有倾斜 | 固定飞行高度，选鲁棒匹配方式 |
| 实时性 | VLM响应1-3秒，无人机已移动 | 低频匹配 + 慢速移动 |
| 搜索范围 | 不知道目标方向 | 螺旋扩展搜索 |
| 图像质量 | 光照、分辨率差异 | 选特征明显的建筑先验证 |

---

## VLN 的真正价值（区别于端到端飞行）

纯坐标导航 = 端到端飞行，不涉及视觉。

VLN 有意义的场景：
- "飞到那个有红色屋顶的建筑旁边"
- "沿着河流向北飞，看到桥就停"
- "找到停车场最空旷的角落降落"
- GPS 丢失时靠视觉定位

**推荐架构：粗导航用坐标，精导航 + 目标识别用视觉。**

client = airsim.MultirotorClient()
